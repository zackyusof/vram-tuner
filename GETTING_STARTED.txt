STEP 1: TRANSFER TO YOUR MAC

Option A: Clone from Gitea (if you have account setup)
  git clone git@git.gonz3.net:zack/vram-tuner.git ~/VRAMTuner

Option B: Transfer via SCP
  scp -r user@linux-server:~/VRAMTuner ~/VRAMTuner

Option C: Manual copy
  - Create folder ~/VRAMTuner on Mac
  - Copy all files from this directory

═══════════════════════════════════════════════════════════════════════

STEP 2: BUILD ON YOUR MAC

Open Terminal on your Mac, then:

  cd ~/VRAMTuner
  make install

That's it! The app will be installed to /Applications/VRAMTuner.app

═══════════════════════════════════════════════════════════════════════

STEP 3: LAUNCH & USE

Find the GPU icon in your menu bar (top right)
Click it → "Show Window"

Or: Cmd+Space → type "VRAM Tuner" → Enter

EXAMPLE USAGE:

On a 32GB Mac:
  - Default GPU allocation: 21.33 GB (66%)
  - Click "Use Recommended": Sets to 24 GB
  - Result: Can run 32B models smoothly
  - Faster inference, longer contexts

On a 64GB Mac:
  - Default GPU allocation: 49.15 GB (75%)
  - Click "Use Recommended": Sets to 56 GB
  - Result: Can run 70B models
  - Use with Ollama, LM Studio, or Dify

TROUBLESHOOTING:

Q: "swift: command not found"
A: Run: xcode-select --install

Q: App won't build
A: Check Swift version (should be 5.9+): swift --version

Q: App asks for password
A: That's normal! It needs sudo to change VRAM. Enter your Mac password.

Q: Want to uninstall
A: rm -rf /Applications/VRAMTuner.app
